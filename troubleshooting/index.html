    Troubleshooting

    Solutions to common issues and error messages.

    Installation Issues

    pip install fails

    Symptom: 

    ERROR: Could not find a version that satisfies the requirement tgdl

    Solutions:

    1. Update pip: 

      # Linux/macOS
python3 -m pip install --upgrade pip

# Windows
python -m pip install --upgrade pip

    2. Check Python version: 

      python --version  # Should be 3.7+

    3. Try with python3: 

      python3 -m pip install tgdl

    Import errors on Ubuntu/Debian

    Symptom: 

    ImportError: cannot import name 'PBKDF2' from 'cryptography.hazmat.primitives.kdf.pbkdf2'

    Solution:

    Update cryptography: 

    pip install --upgrade cryptography

    Or reinstall tgdl: 

    pip install --force-reinstall tgdl

    Permission denied

    Symptom: 

    PermissionError: [Errno 13] Permission denied

    Solution:

    Install in user directory: 

    pip install --user tgdl

    Or use virtual environment (recommended): 

    python -m venv venv
source venv/bin/activate  # Linux/macOS
# venv\Scripts\activate   # Windows
pip install tgdl

    Authentication Issues

    Invalid phone number

    Symptom: 

     Error: PHONE_NUMBER_INVALID

    Solutions:

    1. Include country code: 

       Wrong: 1234567890
 Correct: +11234567890

    2. No spaces or dashes: 

       Wrong: +1 123-456-7890
 Correct: +11234567890

    3. Verify number is registered:

    4. Check you can login to Telegram app with this number

    Verification code issues

    Symptom: 

     Error: PHONE_CODE_INVALID

    Solutions:

    1. Check for typos:
    2. Code is usually 5 digits

    3. No spaces

    4. Don't wait too long:

    5. Code expires after ~10 minutes

    6. Request new code if expired

    7. Check Telegram app:

    8. Code appears in Telegram app (Telegram service message)

    9. Not SMS (usually)

    10. Try SMS code:

    11. Wait for SMS if app code doesn't work
    12. Check spam folder

    Two-factor authentication error

    Symptom: 

    SessionPasswordNeededError

    Solution:

    1. tgdl will prompt for 2FA password: 

      Enter 2FA password:

    2. If you forgot password:

    3. Reset via Telegram app
    4. Settings Privacy and Security Two-Step Verification

    5. Reset password (requires recovery email)

    6. If you don't have 2FA:

    7. This error shouldn't appear
    8. May indicate account security issue

    Session expired

    Symptom: 

     Error: Session expired or invalid

    Solutions:

    1. Logout and login again: 

      tgdl logout
tgdl login

    2. Remove session manually: 

      # Linux/macOS
rm ~/.tgdl/tgdl.session*

# Windows
del %USERPROFILE%\.tgdl\tgdl.session*

    Then login again: 

    tgdl login

    1. Check API credentials: 
      tgdl status
    2. Verify API ID and Hash are correct

    Could not find input entity

    Symptom: 

     Error: Could not find the input entity for PeerUser

    Solutions:

    1. For channels/groups - verify you're a member: 

      # List your channels
tgdl channels

# Check if channel ID is listed

    2. Join the channel:

    3. Open Telegram app
    4. Join the channel/group
    5. Wait a few seconds

    6. Try download again

    7. Use correct entity ID:

    8. From tgdl channels output

    9. Not from web URL (use different format)

    10. For private channels: 

      # Use channel ID from tgdl channels
tgdl download -c 1234567890

# NOT from web URL

    Download Issues

    No files downloaded

    Symptom: 

     No files downloaded.

    Possible Causes & Solutions:

    1. Channel has no media: 
      # Check with limit
tgdl download -c 1234567890 --limit 5

    2. If still zero, channel may only have text

    3. Filters too restrictive: 

      # Try without filters
tgdl download -c 1234567890 --limit 5

# If this works, adjust filters

    4. All files already downloaded: 

      # Check output for:
Found 50 already downloaded files, will skip...
Found 0 new media files to download

    5. This is normal (no new content)

    6. Wrong entity type: 

      # If it's a group, use -g not -c
tgdl download -g 1234567890

    7. Message ID range issue: 

      # Try without range limits
tgdl download -c 1234567890 --limit 5

    Download interrupted

    Symptom: 

    ^C
 Download cancelled by user.

    Solution:

    Run the same command again - tgdl will resume: 

    tgdl download -c 1234567890

    Already downloaded files are automatically skipped.

    Slow download speed

    Possible Causes & Solutions:

    1. Network speed limitation:
    2. Check internet speed

    3. Large files take time

    4. Too many concurrent downloads: 

      # Reduce concurrent count
tgdl download -c 1234567890 --concurrent 3

    5. Telegram server limitations:

    6. Normal during peak hours

    7. Try different time of day

    8. Large file sizes: 

      # Check file sizes
ls -lh downloads/entity_1234567890/

# Filter smaller files
tgdl download -c 1234567890 --max-size 50MB

    Connection timeout

    Symptom: 

    TimeoutError: Connection timed out

    Solutions:

    1. Check internet connection: 

      ping telegram.org

    2. Retry: 

      tgdl download -c 1234567890

    3. Already downloaded files will be skipped

    4. Reduce concurrent downloads: 

      tgdl download -c 1234567890 --concurrent 3

    5. Check firewall/proxy:

    6. Telegram may be blocked
    7. Try VPN if in restricted region

    FloodWaitError

    Symptom: 

    FloodWaitError: A wait of 300 seconds is required

    Cause: Too many requests to Telegram

    Solutions:

    1. Wait the specified time: 

      # Wait 300 seconds (5 minutes)
sleep 300
tgdl download -c 1234567890

    2. Reduce concurrent downloads: 

      tgdl download -c 1234567890 --concurrent 2

    3. Download in smaller batches: 

      # Use limit to control batch size
tgdl download -c 1234567890 --limit 50
# Wait between batches
sleep 300
tgdl download -c 1234567890 --limit 50

    4. Avoid rapid repeated requests:

    5. Don't run multiple tgdl instances
    6. Add delays in scripts

    Duplicate downloads

    Symptom: Files being re-downloaded despite "skipping" message

    Solution:

    This should be fixed in version 1.1.4+. If you're experiencing this:

    1. Update tgdl: 

      pip install --upgrade tgdl

    2. Check version: 

      pip show tgdl

    3. If issue persists:

    4. Report as bug on GitHub
    5. Include: tgdl version, command used, output

    Permission Issues

    Cannot create directory

    Symptom: 

    PermissionError: [Errno 13] Permission denied: '/path/to/downloads'

    Solutions:

    1. Use different output directory: 

      # User's home directory
tgdl download -c 1234567890 -o ~/tgdl_downloads

    2. Check directory permissions: 

      # Linux/macOS
ls -ld downloads/
chmod 755 downloads/

    3. Run without sudo:

    4. Don't use sudo with tgdl
    5. Use user directory instead

    Cannot write file

    Symptom: 

    PermissionError: [Errno 13] Permission denied: 'downloads/12345.jpg'

    Solutions:

    1. Check available disk space: 

      # Linux/macOS
df -h

# Windows
dir

    2. Check file already exists: 

      ls -l downloads/entity_1234567890/12345.jpg

    3. Delete if corrupted

    4. Change output directory: 

      tgdl download -c 1234567890 -o ~/Downloads

    API Credential Issues

    Invalid API credentials

    Symptom: 

     Error: API_ID_INVALID

    Solutions:

    1. Verify credentials:
    2. Go to https://my.telegram.org/apps

    3. Check API ID and Hash

    4. Re-enter credentials: 

      tgdl logout
tgdl login

    5. Enter correct API ID and Hash

    6. Check for typos:

    7. API ID: numbers only
    8. API Hash: 32 character alphanumeric string

    API credentials not found

    Symptom: 

     Error: API credentials not found

    Solution:

    Login first: 

    tgdl login

    This will prompt for API credentials.

    File Issues

    Files not named correctly

    Symptom: Files named generically (photo.jpg) instead of with message ID

    This is expected for: - Old versions (pre-1.1.0) - Single file downloads via download-link

    For regular downloads: - Should be named: {message_id}.{extension} - Example: 12345.jpg

    If incorrect: 1. Update tgdl: 

    pip install --upgrade tgdl

    1. Re-download: 
      tgdl download -c 1234567890

    Corrupted files

    Symptom: Files won't open or are incomplete

    Solutions:

    1. Check file size: 
      ls -lh downloads/entity_1234567890/

    2. Very small files (< 1KB) may be corrupted

    3. Delete and re-download: 

      # Delete corrupted file
rm downloads/entity_1234567890/12345.jpg

# Re-download (tgdl will get missing files)
tgdl download -c 1234567890

    4. Check disk space:

    5. Insufficient space can cause corruption

    6. Download interrupted:

    7. Run download again to resume

    Configuration Issues

    Config file not found

    Symptom: 

     Error: Configuration file not found

    Solution:

    Login to create config: 

    tgdl login

    Cannot read config file

    Symptom: 

     Error: Cannot read configuration file

    Solutions:

    1. Check file permissions: 

      # Linux/macOS
ls -l ~/.tgdl/config.json
chmod 600 ~/.tgdl/config.json

    2. Reset configuration: 

      # Backup first
cp ~/.tgdl/config.json ~/.tgdl/config.json.backup

# Logout and login again
tgdl logout
tgdl login

    3. Manually remove config: 

      # Linux/macOS
rm ~/.tgdl/config.json
rm ~/.tgdl/.key

# Windows
del %USERPROFILE%\.tgdl\config.json
del %USERPROFILE%\.tgdl\.key

# Login again
tgdl login

    Progress Tracking Issues

    Progress not saved

    Symptom: Downloads start from beginning each time

    Solutions:

    1. Check progress file: 

      # Linux/macOS
cat ~/.tgdl/progress.json

# Windows
type %USERPROFILE%\.tgdl\progress.json

    2. Verify file permissions: 

      # Linux/macOS
ls -l ~/.tgdl/progress.json
chmod 644 ~/.tgdl/progress.json

    3. Reset progress: 

      # Backup first
cp ~/.tgdl/progress.json ~/.tgdl/progress.json.backup

# Reset
echo "{}" > ~/.tgdl/progress.json

    Wrong progress tracking

    Symptom: Downloads skip files that weren't downloaded

    Solutions:

    1. Check progress file: 
      cat ~/.tgdl/progress.json

    2. Shows last downloaded message ID per entity

    3. Reset progress for entity: 

      # Edit progress.json
# Remove entry for entity ID

    4. Or reset all progress: 

      echo "{}" > ~/.tgdl/progress.json
tgdl download -c 1234567890

    Command Issues

    Command not found

    Symptom: 

    tgdl: command not found

    Solutions:

    1. Check installation: 

      pip show tgdl

    2. Add to PATH: 

      # Linux/macOS - add to ~/.bashrc or ~/.zshrc
export PATH="$HOME/.local/bin:$PATH"
source ~/.bashrc

# Windows - pip should handle this automatically

    3. Use python -m: 

      python -m tgdl login

    4. Reinstall: 

      pip uninstall tgdl
pip install tgdl

    Invalid command/option

    Symptom: 

    Error: No such option: --xyz

    Solutions:

    1. Check help: 

      tgdl --help
tgdl download --help

    2. Verify spelling: 

       Wrong: --max_size
 Correct: --max-size

    3. Check command structure: 

      # Correct order
tgdl download -c 1234567890 --max-size 100MB

# Wrong order
tgdl --max-size 100MB download -c 1234567890

    Platform-Specific Issues

    Windows: PowerShell encoding issues

    Symptom: Unicode characters display incorrectly

    Solution:

    Set UTF-8 encoding: 

    [Console]::OutputEncoding = [System.Text.Encoding]::UTF8

    Add to PowerShell profile to make permanent: 

    notepad $PROFILE

    Linux: SSL certificate errors

    Symptom: 

    SSLError: certificate verify failed

    Solutions:

    1. Update certificates: 

      # Ubuntu/Debian
sudo apt-get update
sudo apt-get install ca-certificates

# Fedora
sudo dnf update ca-certificates

    2. Update Python certifi: 

      pip install --upgrade certifi

    macOS: notarization warnings

    Symptom: Security warnings when running tgdl

    Solution:

    This shouldn't happen with pip installations. If it does:

    # Reinstall
pip uninstall tgdl
pip install tgdl

    Performance Issues

    High memory usage

    Symptom: System slows down during download

    Solutions:

    1. Reduce concurrent downloads: 

      tgdl download -c 1234567890 --concurrent 2

    2. Download in smaller batches: 

      tgdl download -c 1234567890 --limit 50

    3. Close other applications:

    4. Free up RAM

    5. Close browser tabs

    6. Check system resources: 

      # Linux
top

# macOS
top

# Windows
Task Manager

    High disk usage

    Symptom: Disk space fills up quickly

    Solutions:

    1. Use size limits: 

      tgdl download -c 1234567890 --max-size 50MB

    2. Use download limits: 

      tgdl download -c 1234567890 --limit 100

    3. Check available space first: 

      df -h  # Linux/macOS

    4. Clean old downloads: 

      # Remove files older than 30 days
find downloads -type f -mtime +30 -delete

    Getting Help

    Check logs

    Download logs: 

    tgdl download -c 1234567890 > download.log 2>&1
cat download.log

    System info: 

    # Python version
python --version

# tgdl version
pip show tgdl

# OS info
uname -a  # Linux/macOS
systeminfo  # Windows

    Report bugs

    On GitHub: https://github.com/kavidu-dilhara/tgdl/issues

    Include: 1. tgdl version (pip show tgdl) 2. Python version (python --version) 3. Operating system 4. Full error message 5. Command you ran 6. Expected vs actual behavior

    Ask for help

    Provide: - What you're trying to do - What command you ran - Full error message (if any) - What you've tried so far

    Example: 

    I'm trying to download videos from a channel but getting zero files.

Command: tgdl download -c 1234567890 -v
Output:  No files downloaded.

I can see the channel has videos in Telegram app.
I've tried without filters and still get zero files.

tgdl version: 1.1.4
Python version: 3.9.7
OS: Ubuntu 22.04

    Quick Diagnosis

    Checklist for issues:

    • Latest tgdl version? (pip install --upgrade tgdl)
    • Python 3.7+? (python --version)
    • Logged in? (tgdl status)
    • Member of channel/group?
    • Correct entity flag (-c/-g/-b)?
    • Internet connection working?
    • Enough disk space?
    • Correct API credentials?

    Common fixes:

    # Update tgdl
pip install --upgrade tgdl

# Reset session
tgdl logout
tgdl login

# Test with minimal command
tgdl download -c 1234567890 --limit 3

# Check status
tgdl status
